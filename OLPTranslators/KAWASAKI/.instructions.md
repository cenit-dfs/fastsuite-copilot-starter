# Kawasaki AS-Language Downloader Rules

## Critical: BoolAttribute Returns Python `bool`

`attribute.GetValue()` for BoolAttribute returns Python `True`/`False` (bool type), **NOT** the string `"True"`/`"False"`.

```python
# WRONG ‚Äî always evaluates to False:
if attr.GetValue() == 'True':

# CORRECT:
if str(attr.GetValue()) == 'True':
```

This applies everywhere: `IsComment`, `DLLTOff`, `DLSSDown`, `DLRTPM`, `ExactStop`, `SignalValue` (bool), and any other bool attribute from `GetAttributes()`.

## Euler Convention

- E2 internal: `Euler_XYZs` (static XYZ)
- Kawasaki OAT: `Euler_ZYZr` (rotating ZYZ)
- Always convert using `cenpymath.Euler.Converter.ConvertEuler(rx, ry, rz, Euler_XYZs, Euler_ZYZr)`

## Unit Conversions (E2 ‚Üí AS output)

| Quantity | E2 Unit | AS Unit | Factor |
|----------|---------|---------|--------|
| Position XYZ | m | mm | √ó1000 |
| Speed Contour | m/s | mm/s | √ó1000 |
| Speed PTP | % | % | √ó1 (direct) |
| Accuracy | m | mm | √ó1000 |
| Weave Width | m | mm | √ó1000 |
| LT Bias | m | mm | √ó1000 |
| Joint angles | deg | deg | √ó1 (direct) |

## Buffered Motion Data Pattern

Motion settings (SPEED, ACCURACY, CP) are **buffered** and flushed before each motion via `_flushPendingMotionData()`:

- Set pending values when events fire
- Flush (emit) only when value differs from `last*` tracking variable
- **SPEED suppressed** when `arcOnActive == True` (welder controls speed)
- **ACCURACY suppressed** when `arcOnActive == True`
- **CP**: Only emit `CP OFF` once. Never emit `CP ON`. Never toggle.

## State Tracking

| Variable | Meaning | Set Where |
|----------|---------|-----------|
| `arcOnActive` | Between LWS event and LWE motion output | `_handleArcOn` ‚Üí True, `HandleMotion` (after output) ‚Üí False |
| `arcOnIsFirstMotion` | First motion after ArcOn (suppress LMOVE keyword) | `_handleArcOn` ‚Üí True, `HandleMotion` ‚Üí False |
| `inWeldSection` | Broader weld range (ArcOn to ArcOff) | `_handleArcOn` ‚Üí True, `_handleArcOff` ‚Üí False |
| `explicitMode` | Explicit coordinate output (no CP, no TRSUB) | `ProgramStart` from attributes |
| `rotbaseOn` | TRSUB wrapping active | `OperationStart` based on base profile name |
| `hasTouchSensing` | Emit kr_touch/kr_frame subroutines | Set True when touch events detected |

## TRSUB Wrapping

When `rotbaseOn == 1`, cartesian positions are wrapped: `TRSUB(posName)`.
Exceptions: Joint targets (`#posName`), explicit mode.

## Output File Structure

```
.PROGRAM {name}()
  ; header comments
  TOOL {idx}
  BASE {idx} [/ ROTBASE_ON=1]
  [CP OFF]                    ‚Üê only if not touch/explicit
  HOME
  ... motion/event body ...
  HOME
.END
[kr_touch program]            ‚Üê only if hasTouchSensing
[kr_frame program]            ‚Üê only if hasTouchSensing
.TRANS
  {sorted cartesian positions}
.JOINTS
  {sorted joint positions}
```

## Formatting Helper

Use `_fmtRound1(value)` for numeric output: rounds to 1 decimal, strips trailing `.0`.
- `0.0` ‚Üí `"0"`, `4.6` ‚Üí `"4.6"`, `100.0` ‚Üí `"100"`

## ArcOn vs ArcWeldCondition Output Order

These have DIFFERENT output orders ‚Äî do not mix them up:

**ArcOn**: W1JOBNO ‚Üí SETCONDW1 ‚Üí W2JOBNO+SETCONDW2 ‚Üí SETCONDW3 ‚Üí sensing ‚Üí RTPM  
**ArcWeldCondition**: weave ‚Üí SETCONDW3 ‚Üí RTPM ‚Üí sensing ‚Üí WELDMODE ‚Üí W1JOBNO+SETCONDW1 ‚Üí W2JOBNO+SETCONDW2
