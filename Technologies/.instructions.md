# Technology Script Rules

## Safety
- **DO NOT MODIFY** E2 core technology definitions or built-in scripts.
- **DO NOT MODIFY** E2 site-packages (`cenpydownload`, `cenpyolpcore`, `cenpylib`, `cenpyupload`).
- **DO NOT** install new Python packages unless explicitly approved.

## Folder Structure
Technology scripts follow this layout:
```
Technologies/<TechName>/<Vendor>/Standard/Scripts/
```

Common callback scripts:
- `PostTechInitAttributes.py` — called once when technology is initialized; set up custom attributes
- `PostWmSyncPgAttributes.py` — called when workmethod syncs to program; sync attribute values
- Event handler scripts — handle technology-specific events

## Coding Conventions
- **Logger**: Always a local variable — `logger = operator.GetLogOperator()`. Never store as instance attribute.
- **No debugging prints**: Use E2 logger (`logger.LogDebug()`, `.LogInfo()`, `.LogWarn()`, `.LogError()`).
- **Indentation**: 3 spaces (match workspace default).

## API Context
- **Primary reference**: `docs/API_Python/E2_API_Offline_Programming.md`
- **Core objects**: `docs/API_Python/E2_API_OlpCore.md`

## Attribute Management
- Create attributes with `SetAttribute<Type>()` methods.
- Read attributes with `Get<Type>Attribute()` or generic `GetAttributes()`.
- Use `GetAttributeByName()` when you need `SetVisibility()` — never create with `SetVisibility(False)`.
- For enums, use `GetLiteralAttribute(name, True).GetValue()` — NOT `GetStringAttribute()`.
